# 为什么选择 Burn？

既然已经有 PyTorch、TensorFlow 和其他框架存在，为什么还要费心从零开始创建一个全新的深度学习框架呢？剧透一下：Burn 不仅仅是在 Rust 中复制 PyTorch 或 TensorFlow。它代表了一种新颖的方法，在适当的领域做出正确的权衡，以实现卓越的灵活性、高性能和无缝的开发体验。Burn 不是专门为一种类型的应用程序而设计的框架，它被设计为一个适用于广泛研究和生产用途的多功能框架。Burn 设计的基础围绕三个关键用户群体：

## 机器学习研究者

**机器学习研究者**需要高效的工具来构建和执行实验。对他们来说，能够快速迭代想法并设计可测试的实验至关重要，这些实验可以帮助他们发现新的发现。框架应该促进前沿研究的快速实现，同时确保快速的执行测试。

## 机器学习工程师

**机器学习工程师**是另一个需要考虑的重要群体。他们的重点较少在于快速实现，而更多在于建立稳定性、无缝部署和经济高效的运营。他们寻求可靠、经济的模型，能够在不过度花费的情况下实现目标。从训练到推理的整个机器学习工作流程必须尽可能高效，并具有最少的不可预测行为。

## 底层软件工程师

与硬件供应商合作的**底层软件工程师**希望他们的处理单元能够尽可能快地运行模型，以获得竞争优势。这项工作涉及利用硬件特定功能，如英伟达的 Tensor Core。由于他们主要在系统级别工作，他们希望对计算将如何执行拥有绝对的控制权。

## 目标

Burn 的目标是满足所有这些用户群体的需求！
